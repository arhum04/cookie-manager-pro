# Contributing to Cookie Manager Pro

Thank you for your interest in contributing. This project follows professional software engineering standards.

## Code of Conduct

Maintain a professional and respectful environment in all interactions.

## How to Contribute

1. **Fork the Repository**: Create your own copy of the project.
2. **Branching**: Use descriptive branch names (e.g., `feat/add-search-highlighting` or `fix/modal-overflow`).
3. **Standards**: 
   - Follow the established TypeScript and React patterns.
   - Ensure all changes are covered by tests.
   - Run `npm run build` locally before submitting.
4. **Pull Requests**: Provide a clear description of the change and link to any relevant issues.

## Technical Standards

- **Linting**: Ensure code passes project linting rules.
- **Testing**: All new features require corresponding Vitest integration tests.
- **Documentation**: Update the README or inline comments if changing public APIs or complex logic.
